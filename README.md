# Expense Tracker

Offline-first personal finance app built with Expo (React Native) and FastAPI. All data lives in local SQLite first — the backend is a sync target, not a dependency.

## Stack

| Layer | Technology |
|---|---|
| Mobile | Expo 54 + React Native 0.81 + Expo Router 6 |
| Language | TypeScript 5.9 (strict) |
| Local DB | expo-sqlite + Drizzle ORM |
| State | Zustand (client) + TanStack Query (server/sync) |
| Forms | React Hook Form + Zod |
| Backend | FastAPI (Python) |
| Sync | Custom offline-first sync engine (last-write-wins + tombstones) |

## Prerequisites

- Node 20+
- Expo Go or a physical device / emulator
- Python 3.11+ (for backend — see `/backend`)

## Getting started

```bash
npm install
npm start
```

Press `a` for Android, `i` for iOS, `w` for web.

## Environment

Copy `.env.example` to `.env` and fill in:

```bash
cp .env.example .env
```

| Variable | Description |
|---|---|
| `EXPO_PUBLIC_API_URL` | FastAPI base URL (e.g. `http://localhost:8000`) |

## Project structure

```
app/                  # Expo Router pages (file-based routing)
  (auth)/             # Login / register — unauthenticated guard
  (app)/              # Protected screens
    (tabs)/           # Bottom tab navigator
src/
  components/         # UI primitives and feature components
  db/                 # SQLite schema, migrations, repositories
  store/              # Zustand stores
  api/                # FastAPI client (fetch wrappers)
  hooks/              # Shared custom hooks
  lib/                # Sync engine, conflict resolution, utilities
  types/              # Shared TypeScript types
constants/            # Theme, categories, config
```

## Database migrations

Migrations are generated by Drizzle Kit and run automatically on app launch.

```bash
npx drizzle-kit generate   # generate migration from schema changes
npx drizzle-kit studio     # browse local SQLite in browser
```
